## 2025-04-06 - useLiveClock Singleton Pattern
**Learning:** `useLiveClock` creates a `setInterval` in `useEffect` when consumed. When multiple components like `MissionTimerDisplay` and `MissionDepletingBar` mount it simultaneously, they inadvertently spin up independent interval loops running at high frequencies (1Hz). This leads to uncoordinated re-renders and CPU bloat.
**Action:** Implemented a singleton state wrapper around `useLiveClock` utilizing a `Set` of subscribers. When the first subscriber attaches, the interval starts. When the final subscriber unmounts, the interval clears. Additionally, when testing singleton hooks with Vitest, it requires exposing a test-only teardown helper (e.g. `_resetLiveClockForTesting()`) to wipe module-level state between tests.
