## 2024-05-24 - Cryptographically Secure Randomness

**Vulnerability:** Use of predictable pseudo-random number generators (`Math.random()`) for sensitive values, specifically the `remoteKey` used as an authentication/authorization token for the remote bridge in `electron/remote-bridge.ts`, as well as ID generation in `electron/api.ts` and `src/mission-control/store/useMCStore.tsx`. `Math.random()` lacks the entropy required for security contexts and can be predictably seeded or guessed.
**Learning:** `Math.random()` was used for generating tokens that grant remote control access to the application, creating a critical vulnerability where an attacker could theoretically predict the token and take control.
**Prevention:** Always use cryptographically secure random number generators (CSPRNG) like `crypto.randomBytes()`, `crypto.randomUUID()` in Node.js, or `self.crypto.randomUUID()`/`window.crypto.getRandomValues()` in the browser when generating tokens, keys, passwords, or unpredictable identifiers.
